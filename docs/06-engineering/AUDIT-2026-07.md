## DECISION NEEDED (do not resolve unilaterally)

1. **A3 — the type system (S1/S2).** The product ships **two** type systems: the landing
   renders in **Hanken Grotesk + Spline Sans Mono** (the vendored brand kit at
   `docs/10-design/tacet-brand/`, which reuses the same `--font-sans`/`--font-mono` token
   names with the *other* values), while `docs/10-design/tokens.md`, `typography.md`, and the
   app's `theme.css` still name **Jost + Space Mono** — and the app loads **neither** (A3-3),
   so the five pillars render in system-ui. Per the resolution rule, tokens.md was *not*
   updated to the brand kit, so this is a genuine fork requiring sign-off.
   **Recommendation:** adopt the brand kit (Hanken/Spline) app-wide — it's newer, fuller,
   already shipping on the landing, and reuses the same token names. Then: add the
   Hanken/Spline `@font-face` to the app CSS chain, update tokens.md + typography.md, and
   retire the Jost/Space Mono files + `legacy/styles.css` (A3-4). This also fixes A3-3.

2. **A7-1 — delete `client/src/legacy/` (S3).** 26 files, fully unrouted and tree-shaken
   (no import outside `legacy/`; only two comments in `App.tsx` mention it). It also owns
   `client/src/styles.css` and `client/src/bits.tsx` (dead the moment legacy goes). Also
   resolves A1-5 (protocol words in dormant strings).
   **Recommendation:** delete the whole tree + its two now-dead dependencies. It's documented
   history in git; keeping it only preserves stale strings that could accidentally re-mount.

3. **A7-4 / A7-5 — docs folder collisions (S3).** Two folders numbered `06`
   (`06-decisions`, `06-engineering`); a near-empty `09-product` (one file) beside the
   13-file `01-product`. Renumbering/merging is mechanically a `git mv` + link rewrite but
   touches ~20-40 relative links and changes a canonical doc's home, so it's flagged rather
   than done blind.
   **Recommendation:** merge `09-product/publishing-philosophy.md` → `01-product/` and
   renumber `06-decisions` → a free slot (e.g. `11-decisions`). Safe mitigation applied this
   session instead: README docs-map completeness (see A6/A7 fixes).

4. **A8-5b — Me handle sanitation rule.** `routes/me.ts` stores a user handle by only
   stripping a leading `@` (`replace(/^@/,'')`), so `@anna@mastodon.social` is stored and
   rendered as `@anna@mastodon.social` — reads as a *claimed* federated handle, though it is
   local-only. Whether to **reject** vs **sanitize** federated-looking handles is a product
   decision.
   **Recommendation:** strip anything after the first `@`/domain segment so a local handle
   can't masquerade as a fediverse address; add the guard test (A8-5b) once decided.
